Reasoner Command Tool Manual

Written by Minsu Jang(minsu@etri.re.kr)
Last Update: 2005-11-17

Synopsis

bossam [options] [owl-uris]

[on Unix or Linux]
    bossam = bossam.sh
[on Windows]
    bossam = bossam.bat

  • options: command-line options
  • owl-uris: a list of URIs indicating OWL documents

Description

    bossam launches an OWL inferencing session. It does this by reading in the options and OWL document URIs, creating a new instance of Bossam inference engine, and passing the options and the URIs to the engine instance.

Options

  • -oir owl-inference-rulebase-uri: Specify the location of the rule-base to be used for OWL inferencing.
  • -rb rule-base-uri: Specify the location of the rule-base written in Buchingae to be used for the inference session. One can specify as many rule-bases written in Buchingae as needed by repeatedly specifying -rb rule-base-uri.
  • -rrb rule-base-uri: Specify the location of the rule-base written in RuleML to be used for the inference session. One can specify as many rule-bases written in RuleML as needed by repeatedly specifying -rrb rule-base-uri.
  • -srb rule-base-uri: Specify the location of the rule-base written in SWRL to be used for the inference session. One can specify as many rule-bases written in SWRL as needed by repeatedly specifying -srb rule-base-uri.
  • -srdf swrl-ontology-base-uri: Specify the location of the extended ontology written in SWRL/RDF to be used for the inference session. The extended ontology is a combination of OWL ontology and SWRL rules, as defined in SWRL specification.
  • -n3 ontology-uri: Specify the location of the ontology written in N3 to be used for the inference session. One can specify as many ontologies written in N3 as needed by repeatedly specifying -n3 rule-base-uri.
  • -out serialization-format: Specify the output format. If one does not specify output format, the default output format(Buchingae) is used. Valid serialization-format values are: ruleml for serialization in RuleML, atoml in ATOML(a simple markup language for ground-fact serialization).

Examples

  • bossam http://www.example.com/owl/sample.owl
    • Performs an inference session by applying the internal OWL inferencing rules to the sample.owl, and prints out the result in Buchingae.
  • bossam -rb file:///c:/temp/bizrules.boo -out ruleml http://www.example.com/owl/sample.owl
    • Performs an inference session by applying the internal OWL inferencing rules and bizrules.boo over the sample.owl, and prints out the result in RuleML.
  • bossam -rb file:///c:/temp/bizrules.boo -rb file:///c:/temp/bizpolicy.boo -out atoml
    • Performs an inference session by reading in the two rulebases, and prints out the result in ATOML, a simple markup language for serializing ground predicates.
  • bossam -srb file:///c:/temp/bizrules.swrl -rrb file:///c:/temp/bizpolicy.ruleml -out ruleml
    • Performs an inference session by reading in the two rulebases, one which is written in SWRL and the other in RuleML, and prints out the result in RuleML.
